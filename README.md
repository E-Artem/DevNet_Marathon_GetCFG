# DevNet_Marathon_GetCFG
DevNet Marathon Homework. Day 1

Описание задания:

Дано: сеть из нескольких коммутаторов и маршрутизаторов, все они доступны по своим IP-адресам. 
Все IP-адреса устройств известны. Все коммутаторы и маршрутизаторы работают под управлением ОС IOS или IOS XE.

Необходимо:
1. Собрать со всех устройств файлы конфигураций, сохранить их на диск, используя имя устройства и текущую дату в составе имени файла.
2. Проверить на всех коммутаторах - включен ли протокол CDP и есть ли у процесса CDP на каждом из устройств данные о соседях.
3. Проверить, какой тип программного обеспечения (NPE - No payload encryption или PE - Payload encryption) используется на устройствах и собрать со всех устройств данные о версии используемого ПО.
4. Настроить на всех устройствах timezone GMT+0, получение данных для синхронизации времени от источника во внутренней сети, предварительно проверив его доступность.
5. Вывести отчет в виде нескольких строк, каждая из которых имеет следующий формат, близкий к такому:
Имя устройства - тип устройства - версия ПО - NPE/PE - CDP on/off, X peers - NTP in sync/not sync.

Пример:
ms-gw-01|ISR4451/K9|BLD_V154_3_S_XE313_THROTTLE_LATEST |PE |CDP is ON,5 peers|Clock in Sync
ms-gw-02|ISR4451/K9|BLD_V154_3_S_XE313_THROTTLE_LATEST |NPE|CDP is ON,0 peers|Clock in Sync

--------------------------------------------------------------------------------------------

Скрипт:
- getcfg.py

Входные данные:
- в csv-файле devices.csv через запятую указываются параметры устройств в сети: host, device_type, username, password, secret
- в случае необходимости адрес NTP-сервера сети может быть изменен в переменной NTP_SRV скрипта getcfg.py

Выходные данные:
- создается директория "config_backups", внутри которой записываются конфигурации устройств заданных в файле csv
- в консоль выводится отчет (формат согласно заданию, см. "Пример" выше)
- данные, выводимые в консоль, дублируются в файл "output.txt"
